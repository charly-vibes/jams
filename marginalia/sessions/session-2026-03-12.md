# Session 2026-03-12 — Image Adjust, Code Review, Color Deconvolution

## Overview

Extended the OCR preprocessing pipeline with manual image adjustment controls, performed a comprehensive code review (Rule of 5 Universal), and added Beer-Lambert color deconvolution as an advanced color separation tool.

## Changes

### 1. Image Adjustment Controls

Added a pre-OCR adjustment modal (`openAdjustModal`) with real-time canvas preview:

- **Brightness** (-100 to +100): Linear offset per channel
- **Contrast** (-100 to +100): Standard contrast formula `(259*(C+255))/(255*(259-C))`
- **Threshold** (0-255): Luminance-based binarization (0 = off)
- **Invert**: Flip all channels
- **RGB Channel Isolation**: Toggle individual R/G/B channels (disabled channels output 0)

Shared pixel pipeline extracted into `applyAdjustments(srcData, dstData, params)` — used by both the preview render and the full-resolution `applyAdjustToBlob()`. Preview uses `requestAnimationFrame` debounce to avoid slider jank.

Settings persist in `localStorage` under `marginalia-adjust` and auto-apply to batch captures. An "Adjustments active" hint shows in capture view when non-default settings are saved.

### 2. OCR Pass Refactor

Rewrote the multi-pass OCR to always run all 5 passes (removed early exit on confidence threshold):

| Pass | Preprocessing | PSM | Rationale |
|------|--------------|-----|-----------|
| raw | Upscale only | 3 | Clean photos, auto layout |
| contrast | Auto-levels | 3 | Low contrast / uneven lighting |
| binarize | Otsu's threshold | 6 | Pure B&W, single block |
| block | Upscale only | 6 | Post-it notes, short text |
| sharp | Unsharp mask | 6 | Blurry small text |

Added `openPassPicker()` modal for comparing all results side-by-side. Selection is tracked by index (not text matching) to handle duplicate results correctly. Includes "Adjust & Re-OCR" button that chains: picker → adjust modal → OCR → new picker (recursive Promise resolution).

### 3. Photo Storage

Captured images are now stored in IndexedDB alongside notes (`photoId` foreign key). Notes display thumbnails with async loading (`loadNotePhotos`). Tapping opens a full-size lightbox. Edit modal includes "Re-run OCR" button that opens adjust → OCR → picker → back to edit.

### 4. Rule of 5 Universal Code Review

Systematic 5-stage review found and fixed 16 issues across correctness, edge cases, and code quality:

**Correctness fixes:**
- Pass picker multi-select: selection used text matching, causing all identical-text passes to highlight. Fixed by tracking `selectedIdx`.
- `container.clientWidth` timing: read immediately after `openModal()` returned 0 before layout. Fixed with `requestAnimationFrame` wrapper.

**Edge case fixes:**
- All RGB channels off → solid black image → wasted OCR. Fixed by disabling "Apply & OCR" when `!chR && !chG && !chB`.
- Large image freeze: full-resolution pixel processing on main thread. Fixed by capping `MAX_ADJUST_DIM = 2000`.

**Code quality:**
- Duplicated pixel pipeline between preview and full-res apply → extracted `applyAdjustments()`.
- No debounce on adjust sliders → `requestAnimationFrame` guard.

### 5. Color Deconvolution (Beer-Lambert)

Added advanced color separation using optical density decomposition (Ruifrok & Johnston 2001 method):

**Math pipeline:**
1. `rgbToOD(r, g, b)` — converts RGB to optical density: `OD = -log10(I/255)`
2. `normalizeVec(v)` — unit vector normalization
3. `crossProduct(a, b)` — 3D cross product for orthogonal complement
4. `invert3x3(m)` — cofactor expansion matrix inverse (returns null if singular)
5. `buildDeconvParams(stain1, stain2)` — builds deconvolution parameters from sampled colors

**Two modes:**
- **Single-stain** (1 color sampled): Projects each pixel's OD onto the stain vector and subtracts it. Residual magnitude → grayscale text intensity via `255 * 10^(-magnitude)`.
- **Two-stain** (2 colors sampled): Builds 3×3 stain matrix [stain1, stain2, cross_product], inverts it, extracts the orthogonal complement channel (index 2) as the text signal.

**UI (in adjust modal):**
- Collapsible "Color Separation" section below existing controls
- Tap preview canvas to sample colors (5×5 pixel average from original image data)
- Two color slots with swatch previews and clear buttons
- Live preview updates via existing `applyAdjustments` pipeline
- Channel toggles dimmed when deconv active (bypassed in the pipeline)
- Settings persist with other adjust params

**Integration:** Deconvolution runs as the first step in `applyAdjustments`. When active, it replaces the RGB channel logic, outputting grayscale. Brightness/contrast/threshold/invert then apply on top.

## Implementation Decisions

- Color deconvolution is opt-in (collapsed by default) to avoid confusing users who don't need it
- Sampling reads from `origData` (unmodified image) not the adjusted preview, so sliders don't affect color picking
- 5×5 pixel averaging on sample tap reduces noise from individual pixel variation
- `Math.pow(10, -x)` per pixel is computationally heavy but acceptable: preview canvas is small (~400px wide), full-res apply is one-time before OCR
- Singular matrix (nearly parallel stain vectors) returns null from `invert3x3`, falling back to no deconvolution rather than producing garbage
