# OCR Lab — Session 2026-03-12

## Summary

Initial implementation of OCR Lab, a standalone OCR parameter tuning workbench for book margin photos.

## Implementation Decisions

### Pipeline Architecture
- All 12 preprocessing steps implemented as pure Canvas API operations (no OpenCV dependency)
- Pipeline is synchronous — runs full chain on each preview with debouncing (150ms)
- Preview capped at 1200px, OCR at 2000px for mobile performance
- Processing overlay shown during pipeline runs

### Ported from Marginalia
- `loadImage` pattern (objectURL → Image element)
- Color deconvolution (Beer-Lambert): `rgbToOD`, `normalizeVec`, `crossProduct`, `invert3x3`, `buildDeconvParams`
- Otsu binarization (same histogram-based algorithm)
- Unsharp mask (3x3 box blur, configurable strength)
- Auto-levels (2nd/98th percentile remap)
- OCR language list (17 languages)
- Tesseract.js v5 CDN tag

### New Implementations
- **Shadow removal**: separable max-filter (row-max → column-max) for morphological dilation, followed by median blur for smooth background estimate, then division normalization (pixel/background * 255)
- **Adaptive Gaussian binarization**: separable Gaussian blur (two 1D passes) for performance, per-pixel threshold with configurable C constant
- **Morphology**: erode/dilate operations for opening (remove specks) and closing (reconnect strokes)
- **Median blur**: per-channel median for denoise step
- **Border padding**: white border injection as final pipeline step
- **Profile system**: save/load/delete named configs to localStorage
- **Stain sampling**: click-to-sample on preview canvas with cursor change feedback

### UI Design
- Mobile-first vertical layout, 600px max-width
- Three collapsible `<details>` sections for pipeline groups
- All controls disabled until image loaded
- Tap preview to toggle original/processed view
- Stain sampling mode: click canvas while in sampling mode, shows crosshair cursor

### Tesseract Configuration
- PSM modes: 3 (auto), 4 (single column), 6 (single block), 7 (single line), 8 (single word), 11 (sparse text), 13 (raw line)
- Checked Tesseract.js v5 API — `preserve_interword_spaces` and dictionary penalty params are not reliably supported in the JS port, so omitted

## Issues & Notes
- Median blur uses array sort for each pixel neighborhood — functional but O(n·k²·log(k²)). Acceptable for small kernels (3-7) at preview resolution
- Shadow removal's separable max-filter is O(n·k) per pass, which is fast enough for kernels up to 41
- Adaptive Gaussian blur builds a full 1D kernel each time — could cache, but not needed at v1 preview sizes
